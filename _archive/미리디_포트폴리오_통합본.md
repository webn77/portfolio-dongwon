## S4. PM 코치 철학

**슬라이드 제목:** 내가 생각하는 시니어 PM의 역할

**메인 메시지 (크게)**
> "좋은 PM을 만드는 것이,
> 좋은 제품을 만드는 가장 빠른 길입니다."

**3가지 코칭 원칙**

1. **산출물이 명확한 팀을 만든다**
   - PRD · 배포계획서 · 회의 아젠다 · 회고 템플릿 표준화
   - 신규 PM 온보딩 기간 단축, 의사결정 보고 단계 간소화 경험

2. **데이터로 공통 언어를 만든다**
   - KPI 정의 → 팀 전체 투명 공유 → 주간 Key Metrics 리뷰
   - "느낌"이 아닌 "숫자"로 우선순위를 결정하는 문화 구축

3. **실패를 자산으로 만드는 회고 문화**
   - 완벽한 프로세스보다 회고를 통해 조직에 맞춰가는 것이 핵심
   - 장애·이슈를 팀 학습의 기회로 전환하는 운영 체계 설계
   - 매 주기마다 회고를 통해 문제점을 파악하고, 다음 사이클에서 반드시 개선 — 반복적 성장의 엔진

## S5. 왜 미리디인가

**슬라이드 제목:** 직접 써보고, PM으로서 분석했습니다

**미리캔버스 직접 사용 경험**
> 이 포트폴리오를 준비하면서 미리캔버스를 직접 사용해보며 분석했습니다.
> 처음 쓰는 사람도 빠르게 결과물을 만들 수 있다는 것,
> 그것이 미리디가 1,600만 가입자를 모은 핵심 강점이라고 느꼈습니다.

**PM 시각으로 발견한 미리디의 기회**
- 1,600만 가입자의 행동 데이터가 있지만, 사용층과 결제층이 다른 상태로 각 세그먼트별 페인포인트 전략이 필요한 시점
- 미리클 AI · 비즈하우스 · 디자인허브 등 강력한 서비스 자산이 있지만, 서비스 간 연결고리를 강화하면 차별성 높은 글로벌 플랫폼으로 성장 가능

**핵심 사용층에 대한 인사이트**
- **소상공인·개인사업자**: 빠른 결과물이 필요하지만 반복 사용 동기 부족 → 전환이 핵심 과제
- **학생·크리에이터**: 가입자 46% 학생, 여성 74%, 10~20대 58% — 모바일 진입점과 커뮤니티 연결이 관건이나 실제 결제로 이어지지 않는 구조
- **실제 결제 유저**: 소상공인·마케터·크리에이터 — 재결제와 락인이 핵심
- **B2B 기업 담당자**: 업무 효율성과 비용 절감에 대한 도입 정당성 확보가 전환 열쇠

**내가 미리디에서 만들고 싶은 것**
> "1,600만 유저의 행동 데이터를 바탕으로
> 각 세그먼트를 재정의하여
> 각자에게 맞는 디자인 페인포인트를 AI로 해결하고
> 한 번 결제한 사람이 자연스럽게 다시 결제하게 만드는 플랫폼을 만드는 것"

**미리디를 선택한 이유**
- 21년부터 매년 60% 이상 성장 · 2년 연속 흑자 · 시리즈 B 완료
- 글로벌 확장 시점에서 PM 조직의 깊이를 함께 쌓아갈 수 있는 타이밍
- 결제/플랫폼 도메인 전문성으로 비즈하우스 · 미리클 성장에 직접 기여 가능

## CS1. 스칼라데이터 — 세그먼트 분리 + 프로세스 개선 (S6~S8)

### S6. 문제 정의
**슬라이드 제목:** 데이터도 없고, 프로세스도 없었다

**상황 (Context)**
- 입사 당시 주요 이용층(30~40대 남성)과 3개 고객군 분류는 존재
- 그러나 로밍 서비스를 실제로 견인하는 핵심 고객층에 대한 정의 부족
- Slack 위주 휘발성 소통, 팀별 툴 분산, 배포 주기 불명확
- 장애 발생 시 원인 추적 불가 · CS 문의 반복 발생

**문제 (Problem)**
> "고객은 알고 있는데, 어떤 고객이 비즈니스를 만드는지는 몰랐다"
- 인구통계 기반 세그먼트는 있지만 결제 행동 기반 세그먼트 없음
- 로밍 서비스 핵심 이용 패턴 · 결제 금액대 분포 미파악
- 팀원들이 같은 정보를 공유하지 못하고 있음
- 기존 어드민은 사용성이 불편하고 솔리드 레이아웃 구조로 기능 추가가 어려운 상태 — 빠른 제품 개발에 병목

**내가 맡은 역할**
총괄 PO로 합류 첫 달 — 데이터 체계와 운영 프로세스 동시 설계

---

### S7. 해결 과정
**슬라이드 제목:** 데이터로 고객을 찾고, 프로세스로 팀을 정렬했다

**Track 1. 고객 세그먼트 분리**
1. Mixpanel 퍼널 설계 — 결제 금액 기준 5개 고객군 세분화
2. 핵심 타겟 발견: 도심 택시·배달 운전자 (월 5~20만원 충전)
3. 이 고객을 위한 '타임특가' 정책 기획 → 매출 견인

**Track 2. 업무 프로세스 재설계**
1. 2주 스프린트 기반 4주 정기배포 체계 수립
2. Notion으로 PRD · 배포계획서 · 회의 아젠다 전 과정 통합
3. 매주 Key Metrics 리뷰 세션 신설 → 팀 전체 데이터 공유

**Track 3. 신규 어드민 구축**
1. 기존 어드민 — 솔리드 레이아웃으로 기능 추가·수정에 높은 개발 비용 발생
2. React + Python Django 기반 신규 어드민 완전 재구축
3. UI · API 모듈화 · 템플릿화로 유동적 수정 가능한 구조 설계 → 개발 기간 50% 단축

**핵심 인사이트**
> "완벽한 프로세스보다, 회고를 통해 조직에 맞춰가는 것이 핵심"
> "실제 비즈니스 기여 고객은 데이터를 봐야만 보인다"

## CS3. 스칼라데이터 — BMW Charging 앱 개편 (S11~S12)

### S11. 문제 & 해결
**슬라이드 제목:** 프리미엄 브랜드의 앱이 불편하면 안 된다

**문제**
- 차지비 계정 + BMW 충전카드 인증 분리 → 복잡한 회원가입 이탈 발생
- 실물 충전카드 신청·재발급·결제 수단 변경을 앱 내에서 할 수 없음
- BMW 파트너사 요구사항 + 사용자 편의성 + 운영 효율성 3가지 동시 충족 필요

**해결**
- 회원가입 루트 통합 — 차지비/BMW 계정 하나로 진입
- 지도 하단 즐겨찾기 배치 — 자주 가는 충전소 즉시 접근
- 충전카드 정지 · 재발급 · 결제 수단 변경 모듈 앱 내 구현
- 오프라인 사용 편의성 최우선 · BMW 차량 이미지 적극 활용

**핵심 인사이트**
> "프리미엄 브랜드 파트너 프로젝트는 브랜드 정체성과 사용성을 동시에 설계해야 한다"
> "오프라인 현장 사용 환경을 먼저 상상하고 설계할 것"

### 🎤 세그먼트 A — 학생 = K팝 팬 크리에이터

**재정의가 필요합니다**

> "학생 = 수행평가 PPT 유저"가 아닙니다.
> "학생 = 팬덤 굿즈를 직접 만드는 크리에이터"입니다.

**팬덤 굿즈 제작 행동 패턴**

```
생일카페 개최 준비:
현수막 1장 (14,900원~) + 포토카드 100장 (12,000원~)
+ 컵홀더 + 스티커 + 엽서 → 전부 비즈하우스 상품

콘서트·팬미팅:
응원 피켓 + 슬로건 현수막 + 팬북

SNS 팬페이지 운영:
배너 이미지 + 카드뉴스 + 프로필
```

**지금 이 고객들은 어디서 만드나?**
포토몬, 프린팅팅, 스티커팩토리로 흩어져 있습니다.
미리캔버스 + 비즈하우스로 원스톱 가능한데 연결이 안 됩니다.

**K팝 굿즈 시장 규모**
4대 엔터사(하이브·SM·JYP·YG) 굿즈 매출 올해 1조원 돌파 예상.
이 중 **팬이 직접 만드는 비공식 굿즈 시장**은 별도입니다.

**AI + 크리에이터 참여 구조 제안**

```
【 제작 】
사진 업로드
→ AI 자동 배경 제거
→ K팝 감성 프레임·필터 추천
→ 포토카드 규격 자동 세팅 (55×85mm)
→ 생일카페 현수막 레이아웃 추천
→ 3분 완성 → 비즈하우스 바로 주문

【 크리에이터 수익화 】
잘 만드는 팬 → 템플릿 제작 → 디자인허브 업로드
→ 다른 팬이 구매해서 커스텀
→ 크리에이터 로열티 수취
→ 미리캔버스 내 "K팝 팬 크리에이터 생태계" 형성

【 글로벌 확장 】
K팝은 이미 180개국 팬덤 보유
해외 팬이 미리캔버스에서 디자인
→ 한국 배송 또는 현지 프린트허브 파트너 연결
```

**결제 모델 전환**
기존: 월정액 Pro (학생에겐 비쌈)
제안: **굿즈 제작 건당 결제** — "포토카드 100장 + Pro 1일권 묶음 상품"
→ 첫 결제 경험을 만들고, 반복 구매로 Lock-in

**Canva와의 결정적 차이**
> Canva는 파일을 줍니다. 미리디는 **실물을 줍니다.**
> 이 차이 하나로 K팝 팬 시장에서 Canva는 경쟁자가 아닙니다.

---

### 🏪 세그먼트 B — 개인사업자 = 출판·인쇄 연결이 핵심

**재정의가 필요합니다**

> "개인사업자 = 상품 이미지 만드는 사람"이 아닙니다.
> "개인사업자 = 디자인부터 인쇄 납품까지 원스톱이 필요한 크리에이터"입니다.

내가 만든 작업물이 며칠 후 실물로 배송되어 오는 경험 — 이 경험을 한 번 심어주면 다음 인쇄도, 그 다음 인쇄도 미리디에서 하게 됩니다.

**지금 이 고객의 실제 고통**

```
현재 플로우 (불편):
미리캔버스 로그인
→ 디자인 완성
→ 비즈하우스 별도 로그인 (계정 동일해야 연동)
→ 규격 다시 맞추기
→ 주문 옵션 선택
→ 결제

원하는 플로우:
디자인 완성
→ [인쇄 주문] 버튼 1번
→ 끝
```

**별도 로그인 = 이탈 포인트 1순위**
이 마찰만 제거해도 비즈하우스 연결 전환율이 올라갑니다.

**개인사업자가 실제로 만드는 것들**

```
온라인:  스마트스토어 상품 상세페이지
         쿠팡 썸네일 이미지
         인스타그램 홍보 카드뉴스

오프라인: 전단지 / 포스터 / 현수막
          명함 / 메뉴판 / 스티커
          청첩장 (셀프 청첩장 수요 급증)
```

**이 모든 작업이 "디자인 → 인쇄"로 끝나야 합니다.**

**AI가 해야 할 일**

```
작업 완성 감지
→ AI: "현수막으로 제작하시겠어요?
       비즈하우스에서 오늘 주문하면 내일 받을 수 있어요."
→ 원클릭으로 비즈하우스 주문 진행 (로그인 없이)
→ 규격 가이드 자동 적용
→ 장바구니 이동
```

**핵심 제안: 계정 통합 또는 SSO 연동**
미리캔버스 ↔ 비즈하우스 별도 로그인 제거
→ 마찰 제거만으로 인쇄 주문 전환율 즉시 상승

### 🏢 세그먼트 C — B2B 기업 = 정기 인쇄 수요가 높은 기업·부서

**타겟을 좁혀야 합니다**

> "B2B 전체"가 아닙니다.
> **정기적으로 인쇄 발주가 반복되는 기업·부서**가 첫 번째 타겟입니다.

이들은 정기적으로, 반복적으로, 대량으로 디자인·인쇄 수요가 발생합니다.
지금은 외주 디자인 + 인쇄소를 따로따로 써서 비용과 시간이 과다하게 소모되고 있습니다.

---

**기업·부서의 정기 발주 목록**

```
📅 연간 반복 발주 항목:

[입사 시즌 — 3월/9월]
웰컴키트 패키지
  → 사원증 + 명함 + 환영 메시지 카드
  → 회사 로고 마우스패드 + 노트 + 볼펜
  → 브랜드 텀블러 + 에코백

[상시]
명함 (퇴사·입사·부서이동 발생 시마다)
사내 행사 현수막 + 포스터
판촉물 (전시회·채용박람회용 굿즈)
회사 브랜드 티셔츠·후드

[특정 이벤트]
창립기념일 굿즈
워크숍·MT 기념품
고객사 답례품
```

**지금 담당자가 겪는 고통**

```
현재 프로세스 (복잡):
디자이너 없음
→ 외주 디자인 의뢰 (5~15만원/건)
→ 시안 수정 2~3회 (1~2주 소요)
→ 인쇄소 별도 발주
→ 납기 확인 → 수령 → 배포

비용: 명함 100장 = 디자인 5만원 + 인쇄 1.5만원 = 6.5만원
시간: 최소 1주일

미리캔버스 + 비즈하우스:
브랜드키트 1회 세팅
→ 명함 템플릿에 이름·직함만 바꾸기
→ 비즈하우스 주문 → 익일 배송

비용: 명함 100장 = 12,000원 (디자인 비용 0원)
시간: 10분
```

**기업이 반복 결제하게 만드는 구조**
> 브랜드키트를 한 번 세팅하면, 명함·현수막·행사물 모두 같은 곳에서 주문하게 됩니다.
> 반복 인쇄 수요가 있는 기업일수록 이탈율이 낮고 객단가가 높습니다.
> 이것이 B2B에서 결제 충성도를 만드는 핵심 구조입니다.

## S29. MCP + LLM — 미리디만의 진짜 차별점

**제목:** "말하면 만들어지고, 만들면 배송된다"

---

### 지금 세상이 바뀌고 있습니다

```
기존 디자인 툴의 한계:
사람이 → 툴을 열고 → 직접 조작 → 파일 저장 → 끝

MCP 시대의 변화:
LLM(ChatGPT·Claude·Gemini)에서 자연어로 요청
→ MCP 서버 통해 미리캔버스 API 직접 호출
→ 디자인 자동 생성 → 비즈하우스 자동 주문 → 배송
```

**Canva에는 이 구조가 없습니다.**
Canva는 디자인 툴 안에서만 작동합니다.
미리캔버스가 MCP 서버를 공개하면
**슬랙·노션·HR시스템·이메일 어디서든 디자인·인쇄 요청이 가능**해집니다.

```
400만건의 인쇄 데이터와 사용 사례 깊이가 없으니까요.

지금 움직이지 않으면 Canva가 먼저 자리를 잡습니다.
```

### 글로벌 경쟁력 강화 — 지금 빠진 것들

미리디가 Canva를 글로벌에서 이기려면 기능이 아니라 **인프라와 사용 사례의 현지화**가 필요합니다.

**① 글로벌 진입 벡터 — K팝 팬덤이 이미 열려 있다**

```
K팝 팬덤은 이미 180개국에 분포합니다.
이들은 지금도 굿즈를 만들기 위해 각국의 인쇄소를 검색합니다.

전략:
미리캔버스 글로벌 K팝 팬 대상 → "한국 배송 or 현지 프린트허브"
→ 프린트허브 해외 파트너십 우선 구축 (일본·미국·동남아)
→ K팝 팬이 글로벌 첫 유저가 되고, 소상공인·B2B로 확장

Canva는 이 문화적 레버리지가 없습니다.
```

**② 글로벌 프린팅 파트너 네트워크 구축**

```
현재: 비즈하우스 = 한국 내 인쇄 커머스
과제: 해외 유저가 미리캔버스에서 디자인 → 현지 배송이 안 됨

전략:
현지 인쇄 파트너 API 연동 (MOQ 없는 소량 인쇄 파트너 우선)
→ 일본: 라쿠스루, 프린팅 바바 등
→ 미국: Canva Print 이외 Printful, Printify 연동
→ 동남아: 비즈하우스 직진출 or 현지 파트너 계약

이 네트워크가 완성되면 Canva Print보다 빠른 로컬 배송이 가능합니다.
```

**③ 로컬라이제이션 전략 — 기능 번역이 아닌 사용 사례 현지화**

```
잘못된 글로벌라이제이션:
UI 번역 → 해외 앱 출시 → "글로벌 진출 완료"

올바른 글로벌라이제이션:
JP 시장: K팝 팬 굿즈 + 동인지 커버 + 동인 이벤트 굿즈
US 시장: Small Business Day 포스터 + 파티 초대장 + 팬아트 굿즈
SEA 시장: 소상공인 현수막 + SNS 마케팅 이미지

각 시장의 "인쇄 수요가 높은 사용 사례"를 먼저 정의하고
해당 템플릿·파트너·가격 정책을 현지화하는 것이 핵심입니다.
```

**④ 글로벌 결제·구독 모델 최적화**

```
현재 약점:
- 유료 전환율 0.6% (Canva 9~10%)
- 단일 Pro 구독 → 해외 구매력 다양성 미반영

전략:
건당 결제 모델 (K팝 팬, 소규모 주문) → 첫 결제 허들 최소화
지역별 가격 차등 (PPP 반영) → 동남아·인도 등 진입 가능
B2B Enterprise 별도 플랜 → 기업 도입 단가 상향
```

**⑤ 글로벌 경쟁자 추가 분석 — Canva만이 아닙니다**

| 경쟁자 | 강점 | 미리디 대응 |
|---|---|---|
| Canva | 글로벌 브랜드·AI 기능 | 실물 인쇄 인프라 + 사용 사례 중심 |
