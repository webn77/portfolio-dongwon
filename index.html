PM PHILOSOPHY

내가 생각하는 시니어 PM의 역할

"좋은 PM을 만드는 것이, 좋은 제품을 만드는 가장 빠른 길입니다"

비즈니스 임팩트부터 정의하고 기능을 설계한다

고객 페인포인트를 먼저 찾고, 비즈니스 임팩트를 정의한 뒤 기능을 설계합니다. PRD로 목표·범위·성공 기준을 명문화하고 KPI·우선순위를 팀 전체와 합의한 뒤 개발에 들어가 방향 없는 스펙 낭비를 막습니다.

팀이 스스로 판단하는 환경을 설계한다

의사결정 기준·PRD 프레임·우선순위 템플릿을 팀이 바로 가져다 쓸 수 있는 도구로 만듭니다. 시니어에게 매번 물어보지 않아도 팀원 스스로 판단하고 실행할 수 있는 환경을 설계합니다.

팀의 목표와 소통을 하나로 만든다

PO·디자이너·개발·사업부 사이의 커뮤니케이션을 직접 주도하고 연결합니다. 데이터 기반 KPI를 공통 언어로 삼아 "느낌"이 아닌 "숫자"로 방향을 합의하고, 팀 전체가 하나의 목표를 향해 실행하는 구조를 만듭니다.

실패를 자산으로 만드는 회고 문화

완벽한 프로세스보다 회고를 통해 조직에 맞춰가는 것이 핵심입니다. 장애·이슈를 팀 학습의 기회로 전환하는 운영 체계를 설계하고, 매 주기마다 반복적으로 성장하는 팀을 만듭니다.

9개의 케이스스터디

1
스칼라데이터
모두의충전 · 고객 세그먼트 분리와 타임특가 설계
충전 매출 60억 / +20%↑ 핵심 세그먼트 2개 발견 타임특가 론칭
모두의충전 — EV 충전 플랫폼
도심 택시·배달·트럭 운전자를 위한 전기차 충전 예약·결제 서비스. 스칼라데이터의 핵심 B2C 수익 제품으로, 전국 충전소 네트워크와 운전자 커뮤니티를 연결합니다. 연간 충전 매출 60억 이상, 핵심 타겟 세그먼트 2개(도심 택시·택배·배달) 중심으로 전략을 재정의했습니다.
01 · PROBLEM

인구통계 기반 세그먼트만 존재, 결제 행동 기반 분석 전무. 어떤 고객이 실제 매출을 만드는지 파악이 불가능한 상태.

CUSTOMER · 도심 택시·배달 운전자

충전 대기 시간은 곧 수입 공백이라 빠른 충전이 최우선인데, 요금이 비싸면 경쟁 사업자 기기로 이동해야 한다. 저렴하고 빠른 충전이 가능한 시간과 위치를 알 방법이 없어 매번 비효율이 반복된다.

OPERATOR · 스칼라 서비스팀

어떤 고객군이 실제 매출을 만드는지 행동 데이터가 없어 타겟 마케팅이 불가능했다. 충전 시간대별·위치별 수요 패턴을 몰라 요금 정책을 일괄 적용할 수밖에 없는 상태였다.

02 · ANALYSIS · 고객 세그먼트 분석
버블 크기 = 고객 수 비중 · 라이트·일반 유저는 많지만 충성도가 낮아 매출 기여가 낮았음
03 · FINDING · 핵심 타겟 발견
왜 핵심 타겟인가

도심에 급속충전기가 집중되어 있어 여러 충전 사업자 기기를 혼용해야 하는 고객. 하루 여러 번 반복 충전이 필요해 가격 민감도가 매우 높고, 충전 시간 절약이 곧 수입과 직결됩니다.

타임특가 설계 기준

도심 급속충전기 한정 적용. 퇴근 시간대와 새벽 시간대 — 경쟁이 적고 빠른 충전이 가능한 시간에 낮은 단가를 제공해 핵심 타겟의 재방문을 유도했습니다.

KEY INSIGHT · 배운 점

실제 비즈니스 기여 고객은 데이터를 봐야만 보인다. 고객 수가 많아도 충성도가 낮으면 매출로 이어지지 않는다. 세그먼트 분리 없이는 핵심 타겟에게 맞는 가격 정책을 설계할 수 없다.

MIRIDIH APPLY · 미리디 적용 포인트

미리캔버스 1,600만 유저 중 실제 유료 전환을 이끄는 충성 세그먼트를 데이터로 발견하고, 세그먼트별 전환 트리거와 가격 정책을 설계하는 데 이 경험을 직접 적용할 수 있습니다.

01 · PROBLEM

포인트 유효기간 정책이 없어 미사용 포인트가 장기 누적되고, 이를 기업 부채로 인식해야 하는 구조였습니다. 고객 입장에서도 소멸 압박이 없어 재방문 동기가 사라졌고, KT 멤버십으로 전환하려면 번거로운 쿠폰 발급 방식을 거쳐야 했습니다.

CUSTOMER · 충전 유저

포인트가 언제 소멸되는지 알 수 없어 사용 동기가 생기지 않는다. KT 멤버십으로 전환하려면 쿠폰을 별도로 발급받아야 하는 번거로움 때문에 포기하는 경우가 많다.

OPERATOR · 스칼라 서비스팀

미사용 포인트가 수억 원 규모로 누적되어 재무적으로 부채로 계상되는 구조. 포인트를 사용하지 않는 비활성 유저가 늘어날수록 재방문율이 낮아지는 악순환이 지속됐다.

03 · SOLUTION · 4가지 재설계
①
IA 전면 재수립 — 5개 파트로 구조화

파편화된 기능을 회원·결제·상품·정산통계·서비스 5개 파트로 재분류. 각 파트가 독립적으로 운영되면서 서로 연계되는 구조를 설계했습니다.

②
시계열 통계 시각화 — 개발팀 의존 제거

일·주·월·년 단위 시계열 통계를 어드민에서 직접 조회·다운로드 가능하도록 구현. 매번 개발팀에 요청하던 데이터 추출을 운영팀이 독립적으로 처리합니다.

③
운영 DB 직접 접근 차단 + 성능 안정화

운영팀이 운영 DB에 직접 쿼리하던 관행을 차단하고 전용 읽기 레이어를 구성. 성능 이슈와 데이터 무결성 문제를 동시에 해결했습니다.

④
다중 매입사 정산 통합 UI

여러 매입사(카드사·지자체)별로 분산된 정산 정보를 하나의 화면에서 조회·처리 가능하도록 통합. 정산팀의 월별 결산 작업 시간을 크게 단축했습니다.

04 · RESULT
개발팀 데이터 요청 대폭 절감

운영팀 셀프 데이터 조회로 개발팀 리소스 확보. 부서 간 대기 시간 제거.

KEY INSIGHT · 배운 점

많은 기능 개선보다 확장성에 초점을 두고 만들어야 한다. 기술 부채는 일찍 해결할수록 좋다. 내부 시스템 설계도 결국 사용자 인터뷰에서 시작해야 진짜 Pain Point를 잡을 수 있다.
